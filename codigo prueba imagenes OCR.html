# funcion para tratar las imagenes

def image_process(img, n1, n2):
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY) # transformar las imagenes a escala de grises

    ret, thresh1 = cv2.threshold(gray, 0, 255, cv2.THRESH_OTSU | cv2.THRESH_BINARY_INV) # crear un recuadro para el recorte de la imagen
    
    # cv2.threshold() tiene 4 parmetros, el primer parmetro es la imagen con cambio de espacio de color, 
    # seguido del valor de umbral mnimo, el valor de umbral mximo y el tipo de umbral que debe aplicarse.
    rect_kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (n1, n2)) # 
    
    # cv2.getStructuringElement() se usa para definir un elemento estructural como elptico, circular, rectangular, etc.
    dilation = cv2.dilate(thresh1, rect_kernel, iterations = 1)
    
    # cv2.findContours() se utiliza para buscar contornos en la imagen dilatada.
    _,contours, hierarchy = cv2.findContours(dilation, cv2.RETR_EXTERNAL,
                                                 cv2.CHAIN_APPROX_NONE)

    image_copy = img.copy()

    return image_copy, contours, hierarchy

# funcion para extraer el texto de las imagenes previamente tratadas
def get_text(image, contours):
    ll = []

    for cnt in contours:
        # coordenada X, coordenada Y, ancho, alto
        x, y, w, h = cv2.boundingRect(cnt)
        
        # Dibujar un rectngulo en la imagen copiada
        rect = cv2.rectangle(image, (x, y), (x + w, y + h), (0, 255, 0), 2)
        
        # Recortar el bloque de texto para dar entrada a OCR
        cropped = image[y:y + h, x:x + w]
        
        
        # Aplicar OCR en la imagen recortada
        text = pytesseract.image_to_string(cropped)
        
        # print(text)
        ll.append(text)
        # Appending the text into file
        # file.write(text)
        # file.write("\n")
        
        # # Close the file
        # file.close
    return ll

Modelado: Descripción de la Estrategia de Análisis. ¶

Todas las imgenes tienen un espectro colores en la escala RGB, conformada por tres matrices con valores numricos, y dependiendo de los valores numricos en esas tres matrices, es que se configura la forma de la imagen, y principalmente, se configura la coloracin; se pueden extraer de ah la cantidad de pixeles por los cuales est conformada una imagen. Estas imgenes se pueden pasar a una escala grises, en la cual la imagen quedara a blanco y negro. Al momento de pasar la imagen a escala de grises, se pasa de tener 3 matrices en la escala RGB (Rojo, verde, azul por sus iniciales en ingls), a 2 matrices, en la cual se configura una matriz para la coloracin clara o la intensidad del color blanco, y otra para la intensidad del color negro.

Gracias a la ayuda de la librera numrica de Python Numpy, se puede contar la cantidad de pixeles de color oscuro, y la cantidad de pixeles claros.

Cmo puede ayudar esto?

Si los documentos que se escanean contienen texto principal, que no sean marcas de agua ni dems, al momento de pasar la imagen a escala de grises, ese texto sern pixeles con valores netamente negros, o con el valor 0, y los espacios en blanco sern representados por pixeles blancos, o con el valor 255. La hiptesis para entrenar el modelo es que los documentos escaneados que no tienen contenido, o que estn en blanco, tienen menos pixeles, principalmente menos pixeles negros, a comparacin de los documentos que tienen contenido. Al tener el recuento de la cantidad de pixeles, se puede entrenar un modelo clasificatorio, para que aprenda la diferencia entre documentos en blanco y documentos con contenido, gracias a la contabilizacin de los pixeles en las imgenes.
